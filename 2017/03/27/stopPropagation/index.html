addEventListener及其事件传播

发表于 2017-03-27 |

addEventListener的参数

addEventListener方法是JavaScript中常用的注册事件方法，调用形式为element.addEventListener(event, function, useCapture)

第一个参数event是不带”on”前缀的事件名

第二个参数function是事件的响应函数，默认会将事件对象作为参数隐式地传入函数中

第三个参数useCapture控制事件的执行顺序：默认false，冒泡执行，事件的响应顺序为“从子元素到父元素，从具体到不具体”；设为true表示捕获执行，事件的响应顺序为“从父元素到子元素，从不具体到具体”

为子元素和父元素都设置点击事件响应函数，输出一条语句，点击子元素的情况下：

  • 第三个参数为false的情况下冒泡执行，事件响应顺序为”son”->”father”.
  • 第三个参数为true的情况下捕获执行，事件响应顺序为”father”->”son”.

阻断事件传播

下图抽象了一棵树的层次

节点层次为

|- root

|—– second-level

|———– third-level

|—————- fourth-level

|———————– fifth-level

现要求点击某个节点，被点击节点高亮显示

//为被点击的节点注册上色事件
function setClickedNodeColor(){
    var root=document.getElementById("root"); //根节点
    root.addEventListener("click",function(){
        setColor3.call(this,this);
    },false);

    var nodes=root.getElementsByTagName("div");
    for(var i=0;i<nodes.length;i++){ //为子节点注册点击事件
        if(nodes[i].className=="second-level" || nodes[i].className=="third-level" || 
            nodes[i].className=="fourth-level" || nodes[i].className=="fifth-level"){
            nodes[i].addEventListener("click",function(){
                setColor3.call(this,this);
            },false);
        }
    }
}
//给被点击的节点上色
function setColor3(node){
    node.style.backgroundColor="#75F2C8";
}

当点击任何节点（非根节点）时所有节点都高亮显示

控制台显示的事件响应顺序也符合冒泡执行的顺序:“从子元素到父元素，从具体到不具体”

要达到点击子元素则只有子元素的点击事件被响应，即阻断子元素之后事件的传播，需要用event.stopPropagation()方法，event是addEventListener方法中执行函数默认传入的事件对象。

此时再点击子节点则父节点的事件不会被响应，事件的传播被中断了
