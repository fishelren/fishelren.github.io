关于CSS3 3D transform的一些笔记

发表于 2017-04-15 |

transform中rotateX，rotateY，rotateZ的正方向：CSS3的坐标系从左到右X轴从负到正，从上到下Y轴从负到正（ps:这与WebGL不同，WebGL中Y轴与其相反），从外到里Z轴从正到负；把各个轴的正方向想象成迎面向你射来的一支箭，顺时针方向对应rotateX，rotateY，rotateZ中的正度数，逆时针方向对应负度数。

不能将包裹多个参与3d变换子元素，并且自身也参与3d变换的包裹元素作为透视对象（镜头观察的整个范围/设置perspective的对象），这样在perspective一定的情况下可能会造成镜头的忽远忽近，产生预料之外的效果。所以，要么就将舞台元素（包裹3d变换子元素，但自身不参与3d变换的元素）设置为透视对象，要么就将单个的参与3d变换的元素设为透视对象。两种方式设置perspective的方法不同。

把自己的眼睛想象成照相机的镜头，perspective就表示镜头距元素平面z=0的距离 ，没有perspective无法产生3D效果（要产生3D效果必须设置perspective）；perspective-origin可以想象成镜头在元素平面上的垂足的位置，默认在透视元素的中心，就好像你站在透视元素的正前方看着它一样。

transform-style: preserve-3d; /* Firefox */
-webkit-transform-style: preserve-3d; /* Chrome、Safari 和 Opera */

设置内嵌的元素（子元素）在 3D 空间如何呈现。有两个值：

  • flat：所有子元素在 2D 平面呈现。（默认值）
  • preserve-3d：保留3D空间。
backface-visibility:visible; /*Internet Explorer 10+ 和 Firefox*/
-webkit-backface-visibility:visible;    /*Opera 15+ / Chrome 和 Safari */
-moz-backface-visibility:visible;     /* Firefox */
-ms-backface-visibility:visible;     /* Internet Explorer */

backface-visibility决定旋转元素背面是否可见

  • visible 背面是可见的。（默认值）
  • hidden 背面是不可见的。

在博主高版本的IE11上都不支持transform 3d变换，IE，你可以！
