gulp使用简介

发表于 2018-01-02 |

gulp.js是基于Node.js开发的一款自动化工具，作用是使重复的流程化工作按任务的方式自动执行。它使用Node.js的stream来读取和操作数据，速度很快。所有的gulp代码编写都可以看做是将规律转化为代码的过程。

全局安装

npm install -g gulp

全局安装gulp后，还需要在项目文件夹中本地安装一次。全局安装是为了在任意地方使用插件命令，本地安装是为了保证全局插件版本不影响到项目插件版本，且本地安装的模块可以通过require引入，而require引入不了全局安装的模块。

在项目文件夹下执行命令：

npm install --save-dev gulp //将gulp作为项目开发阶段的依赖写入package.json中

offsetLeft和offsetTop

发表于 2017-05-04 |

offsetLeft/offsetTop属性是元素相对于其offsetParent属性指定的包含元素而言的，offsetParent不一定等于父节点（parentNode），例如：

/*CSS*/
.box1{
    display: inline-block;
    width:300px;
    height: 300px;
    background-color: #FEA4A4;
}
.box2{
    display: inline-block;
    width:200px;
    height: 200px;
    background-color: #9789FF;
}

<!-- HTML -->
<div class="box1" id="box1">
    <div class="box2" id="box2"></div>
</div>

此时，box1和box2的offsetParent都是body；但如果给box1指定position属性为”relative”，”absolute”或”fixed”，则box2的offsetParent为box1，这说明offsetParent是离元素最近的position属性指定为”relative”，”absolute”或”fixed”的包含元素或初始包含块（一般是body）。

JavaScript中的变量提升

发表于 2017-04-28 |

JavaScript中除了有函数声明提升，还有变量提升。但变量提升只对var有效，对let是无效的。而且这种变量提升是一种“不完全”提升，为什么说是“不完全”提升呢？来看下面的例子：

console.log(x); //undefined
var x=1;

可以看到打印x的时候并没有报未定义（ReferenceError）的错，声明的确是提升了；但是在定义语句前引用var定义的变量得到的值是undefined，说明变量赋值未提升，这是“未完全”

console.log(x); //ReferenceError
let x=1;

可见，let就不存在变量的提升，必须在用let定义变量后再使用变量

这只是概念上的一些理解，实际应用中我们还是“定义在前，使用在后”吧！

关于CSS3 3D transform的一些笔记

发表于 2017-04-19 |

transform中rotateX，rotateY，rotateZ的正方向：CSS3的坐标系从左到右X轴从负到正，从上到下Y轴从负到正（ps:这与WebGL不同，WebGL中Y轴与其相反），从外到里Z轴从正到负；把各个轴的正方向想象成迎面向你射来的一支箭，顺时针方向对应rotateX，rotateY，rotateZ中的正度数，逆时针方向对应负度数。

不能将包裹多个参与3d变换子元素，并且自身也参与3d变换的包裹元素作为透视对象（镜头观察的整个范围/设置perspective的对象），这样在perspective一定的情况下可能会造成镜头的忽远忽近，产生预料之外的效果。所以，要么就将舞台元素（包裹3d变换子元素，但自身不参与3d变换的元素）设置为透视对象，要么就将单个的参与3d变换的元素设为透视对象。两种方式设置perspective的方法不同。
